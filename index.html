1
AI에게 말 걸기
어떻게 지시하고, AI는 무엇을 본다
라틴어 promptus — "앞에 꺼내 놓인, 준비된"
🎭
Prompt
프롬프트
연극에서 배우가 대사를 잊으면 무대 뒤에서 귓속말로 알려주는 사람을 '프롬프터'라고 한다. AI에게 건네는 지시문도 그 귓속말이다.
AI에서는
AI에게 건네는 모든 지시·질문. 어떻게 쓰냐에 따라 결과가 완전히 달라진다.
System = 체계·운영 / 사용자 눈에 안 보이는 무대 뒤
🎬
System Prompt
시스템 프롬프트
영화 촬영 전 감독이 배우에게 "넌 냉정한 탐정 캐릭터야"라고 미리 지시하는 것. 사용자가 말을 걸기도 전에, 숨겨진 곳에서 AI의 성격과 역할을 세팅한다.
AI에서는
사용자에게 보이지 않는 사전 지시문. 챗봇·서비스의 성격을 여기서 만든다.
라틴어 contextus — "함께(con) 엮어짠(texere) 것"
🪟
Context Window
컨텍스트 윈도우
창문 너머 풍경을 보듯, AI는 창 크기만큼만 대화를 볼 수 있다. 창이 작으면 오래된 내용이 밖으로 밀려나 — AI가 "잊는" 것처럼 보이는 이유다.
AI에서는
AI가 한 번에 볼 수 있는 대화의 최대 범위. 초과하면 앞 내용이 사라진다.
라틴어 tokenum — "표시, 증표" → 지하철 토큰·티켓
🎫
Token
토큰
지하철 토큰처럼 AI가 처리하는 최소 단위다. 단어 하나가 한 토큰이 아니라 — "안녕하세요"는 여러 조각으로 쪼개진다. 컨텍스트 윈도우 크기도 토큰 수로 잰다.
AI에서는
텍스트를 처리하는 기본 단위. 비용과 속도가 모두 토큰 수에 비례한다.
2
AI가 답을 만드는 방식
다음 단어를 고르는 메커니즘, 그리고 부작용
Temperature — 물리학에서 입자의 활동성
🌡️
Temperature
온도
얼음(0°)은 형태가 고정되고 예측 가능하다. 수증기(높은 온도)는 자유롭게 퍼지지만 불규칙하다. AI의 창의성도 온도 다이얼 하나로 조절된다.
AI에서는
낮으면 → 안전하고 일관된 답변. 높으면 → 창의적이지만 엉뚱할 수 있음.
Top K = 상위 K개 / Top P = 누적 확률 P%까지
🎯
Top K / Top P
탑 케이 / 탑 피
식당에서 메뉴를 고를 때 —
Top K: "베스트 5 메뉴 중 골라"
Top P: "고객 선택 합계 80%에 드는 것들 중 골라"
온도가 "얼마나 과감하게"라면, Top K/P는 "후보를 몇 개나 볼 건지"다.
AI에서는
다음 단어 후보군의 폭을 설정하는 파라미터. 온도와 함께 출력 다양성을 제어.
라틴어 hallucinari — "헛것을 보다, 방황하다"
👻
Hallucination
할루시네이션
AI는 "가장 그럴듯한 다음 단어"를 계속 고르는 구조다. 정답이 없어도 자연스러운 문장은 만들어진다 — 그게 문제다. 사막에서 목마른 사람이 없는 오아시스를 보는 것처럼, 자신있게 지어낸다.
AI에서는
없는 사실을 그럴듯하게 만들어내는 현상. 팩트 확인은 항상 사람이 해야 하는 이유.
3
AI를 내 것으로
특정 용도에 맞게 훈련·확장하기
Fine-tuning — 악기를 미세하게 조율하다
🎻
Fine-tuning
파인튜닝
피아노를 처음 만드는 게 아니라 — 이미 만들어진 피아노의 줄을 미세하게 조여 내 공연장 음향에 맞게 튜닝하는 것. 기존 AI 모델을 내 회사 데이터로 추가 훈련시킨다.
AI에서는
기존 모델에 특정 데이터를 추가 학습시켜 특정 역할에 최적화하는 과정.
Multi(여러) + Modal(방식, 양식) — 여러 감각 채널
👁️‍🗨️
Multimodal
멀티모달
사람은 눈으로 보고, 귀로 듣고, 글을 읽으며 동시에 이해한다. 멀티모달 AI는 텍스트·이미지·음성을 동시에 처리할 수 있다. "이 사진에서 뭐가 보여?" 같은 질문이 가능해지는 이유.
AI에서는
텍스트 외에 이미지, 음성, 영상 등 여러 입력 형식을 동시에 처리하는 능력.
4
AI가 직접 행동한다
스스로 판단하고, 도구를 쓰고, 협력하기까지
라틴어 agere — "행동하다" → 대리인, 요원
🕵️
Agent
에이전트
제임스 본드에게 "악당 제거"라는 목표만 주면 알아서 이동 수단을 찾고, 무기를 선택하고, 계획을 세운다. 에이전트 AI도 목표만 받으면 도구를 골라 스스로 행동한다.
AI에서는
목표를 받으면 필요한 도구를 스스로 선택하고 단계적으로 실행하는 AI.
Tool Call — 도구(Tool)를 호출(Call)하다
🔧
Tool Call
툴 콜
요리사가 재료를 손질할 때 칼을, 볶을 때 프라이팬을 필요에 따라 집어든다. 에이전트도 검색이 필요하면 검색 도구를, 계산이 필요하면 계산기를 호출해 쓴다.
AI에서는
AI가 외부 기능(검색, 계산, 파일 저장 등)을 필요할 때 꺼내 쓰는 행위.
Orchestra — 여러 악기를 지휘해 하나의 음악으로
🎼
Orchestration
오케스트레이션
바이올린, 첼로, 트럼펫이 각자 잘해도 지휘자 없이는 소음이 된다. 오케스트레이션은 여러 AI 에이전트가 제각각 움직이지 않도록 흐름을 설계하고 조율하는 것이다.
AI에서는
여러 에이전트의 역할·순서·협력을 설계하고 총괄 조율하는 것.
5
AI가 문서를 검색해 답한다
RAG 파이프라인 — 준비에서 검색까지
Retrieval-Augmented Generation — 검색으로 보강한 생성
📚
RAG
래그
오픈북 시험처럼 — 모든 걸 외우는 대신 시험 중 책을 펼쳐 관련 페이지를 찾아 답하는 것. 아래 단계들이 모두 합쳐져 이 방식을 구현한다.
AI에서는
질문이 들어오면 관련 문서를 검색해 그것을 근거로 답변을 생성하는 방식.
Chunk — 덩어리, 조각 / 큰 것을 나누는 행위
🍕
Chunking
청킹
피자 한 판을 통째로 먹을 수 없으니 조각으로 나눠 먹는다. 500페이지 보고서를 AI에게 통째로 줄 수 없으니 소화 가능한 크기로 잘라 처리한다.
AI에서는
긴 문서를 AI가 처리할 수 있는 크기로 나누는 첫 번째 준비 작업.
Embed — 박아 넣다, 심다 / 지도에 핀을 꽂듯
📍
Embedding
임베딩
서울과 부산은 지도 위 다른 좌표에 있다. "사랑"과 "애정"은 의미가 비슷하니 임베딩 공간에서 가까운 좌표에 놓인다. 청킹으로 잘라낸 조각들을 각각 숫자 좌표로 변환한다.
AI에서는
텍스트를 다차원 숫자 좌표로 변환하는 것. 의미가 비슷할수록 좌표가 가깝다.
Vector — 방향과 크기를 가진 화살표 / Database — 창고
🗄️
Vector Database
벡터 DB
일반 창고는 이름으로 물건을 찾는다. 벡터 DB는 "이것과 비슷하게 생긴 걸 찾아줘"라고 하면 찾아준다. 임베딩된 좌표들을 저장하고, 가장 가까운 것을 순식간에 꺼내오는 특수 창고다.
AI에서는
임베딩 벡터를 저장하고, 유사도 기반 검색을 고속으로 처리하는 데이터베이스.
Index — 색인, 목차 / 책 뒤의 찾아보기
📑
Indexing
인덱싱
도서관 사서가 새 책이 들어오면 분류하고 카드에 기록해 나중에 빠르게 찾을 수 있게 준비해두는 것. 청킹 → 임베딩 → 벡터 DB 저장까지를 한데 묶어 인덱싱이라 한다.
AI에서는
문서를 청킹 → 임베딩 → 벡터 DB 저장까지 마쳐두는 사전 준비 과정.
Semantic — 의미론적 / 글자가 아닌 뜻을 본다
🔍
Semantic Search
시멘틱 서치
일반 검색은 "강아지"를 치면 "강아지"가 포함된 글만 찾는다. 시멘틱 서치는 "강아지"를 쳐도 "반려견", "puppy", "멍멍이"가 담긴 글을 찾는다. 인덱싱된 벡터 DB에서 의미 유사도로 검색한다.
AI에서는
글자 일치가 아닌 의미의 유사도로 문서를 검색하는 방식.
Retriever — "되찾아오는 것" / 사냥개 품종 이름
🐕
Retriever
리트리버
골든 리트리버는 사냥꾼이 쏜 사냥감을 물어오도록 훈련된 견종이다. AI에서 리트리버는 사용자 질문을 받으면 벡터 DB에 뛰어들어 가장 관련 있는 청크를 물어온다.
AI에서는
시멘틱 서치로 찾아낸 문서 조각을 벡터 DB에서 실제로 꺼내오는 역할.
Grounding — 접지(接地) / 땅에 연결해 안전하게
⚡
Grounding
그라운딩
전기가 땅(접지선)에 연결되지 않으면 감전 위험이 있다. 리트리버가 물어온 문서를 근거로 AI 답변을 묶어두지 않으면 할루시네이션이 튄다. RAG 파이프라인의 마지막 안전장치다.
AI에서는
AI 답변이 실제 문서·데이터에 근거하도록 묶어두는 것. 할루시네이션 방어막.
